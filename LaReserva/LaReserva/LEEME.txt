================================================================================
  LA RESERVA — Instalación en el PC del restaurante
================================================================================

CONTENIDO DE ESTA CARPETA
-------------------------
  api\          Servidor NestJS (dist, node_modules, vendor, prisma)
  web\          App web exportada (index.html, _expo, etc.)
  apk\          Instalable Android (opcional)

  EN LA RAIZ (uso diario):
  abrir-la-reserva.vbs         Abrir la app sin ventanas CMD (recomendado)
  crear-accesos-directos.vbs   Accesos directos en el escritorio
  inicio.bat / detener.bat     Atajos → carpeta bin\
  icon.ico                     Icono de la app
  LEEME.txt                    Este archivo

  CARPETA bin\ (herramientas y soporte):
  inicio.bat, detener.bat, detener.vbs, mostrar-id-maquina.bat
  evitar-suspension.bat, abrir-firewall.bat, reparar-api.bat
  ejecutar-baseline-prisma.bat, ejecutar-fix-cobro-parcial.bat
  (ver bin\LEEME.txt)

  CARPETA scripts\ (PowerShell interno; no hace falta abrirla a mano)

  Si falta api\dist\main.js o web\index.html:
  • En desarrollo: npm run la-reserva:empaquetar
  • En el restaurante: copia la carpeta LaReserva\ completa.


REQUISITOS (solo una vez en el PC)
----------------------------------
  • PostgreSQL instalado y con una base creada (ej. base "lareserva").
  • Node.js LTS instalado desde https://nodejs.org (siguiente → siguiente).


CONFIGURACIÓN DEL API
---------------------
  En la carpeta api\ debe existir un archivo .env con, como mínimo:

    DATABASE_URL="postgresql://USUARIO:CONTRASEÑA@localhost:5432/NOMBRE_BD?schema=public"
    JWT_SECRET="una-cadena-larga-y-aleatoria"
    PORT=3000
    HOST=0.0.0.0
    NODE_ENV=production

  Ajusta USUARIO, CONTRASEÑA y NOMBRE_BD a tu PostgreSQL.
  HOST=0.0.0.0 permite que celulares en la Wi-Fi lleguen al API usando la IP del PC.
  NODE_ENV=production activa la licencia obligatoria (ver siguiente sección).


LICENCIA (anclada a este PC)
----------------------------
  El sistema solo arranca en el PC autorizado. No se puede copiar la carpeta
  a otro equipo ni alterar el archivo de licencia.

  Activación (una vez por PC):
  1. Doble clic en bin\mostrar-id-maquina.bat
  2. Copia el "ID completo" y envíalo al proveedor del software.
  3. Recibirás un archivo license.key
  4. Colócalo en la carpeta api\ (junto a .env y package.json)
  5. Inicia La Reserva con inicio.bat o abrir-la-reserva.vbs

  Al actualizar LaReserva, conserva api\license.key y api\.env (no los borres).

  Si mueves la instalación a otro PC, la licencia deja de valer: hay que
  solicitar una nueva con el ID de la máquina nueva.


IMPRESORA COMANDA (opcional, Jaltech 58M u otra ESC/POS 58 mm)
----------------------------------------------------------------
  Conecta la impresora al PC del restaurante (USB o Bluetooth emparejado).
  En api\.env agrega (nombre exacto como aparece en Windows):

    PRINTER_ENABLED=true
    PRINTER_INTERFACE=printer:NombreDeTuImpresora
    PRINTER_WIDTH=32

  Si Bluetooth usa puerto COM:  PRINTER_INTERFACE=COM3,COM4
  Si USB (nombre en Windows):   PRINTER_INTERFACE=printer:POS-58
  Prueba desde services/api:    npx tsx scripts/test-print-comanda.ts
  El mesero pulsa «Pasar a cocina» en la app; el API imprime solo platos
  (sin bebidas ni precios). El cobro final incluye todo el pedido.

USUARIOS INICIALES (AUTOMÁTICO)
-------------------------------
  En cada inicio, el sistema verifica datos base sin borrar nada:
  - Roles: mesero, chef, admin
  - Mesas: 1-15, 98 y 99
  - Usuarios demo (solo si no existen):
      mesero@lareserva.local / mesero123
      chef@lareserva.local   / chef123
      admin@lareserva.local  / admin123


ARRANQUE
--------
  OPCIÓN A — Sin ventanas (recomendado en el PC del restaurante)
  1. PostgreSQL en marcha y base creada (ej. la_reserva).
  2. Edita api\.env con DATABASE_URL y JWT_SECRET.
  3. Activa la licencia: bin\mostrar-id-maquina.bat → envía el ID → coloca
     api\license.key (solo la primera vez en este PC).
  4. (Recomendado) Doble clic en bin\evitar-suspension.bat.
  5. Doble clic en crear-accesos-directos.vbs (solo la primera vez) → iconos en escritorio.
  6. Doble clic en "La Reserva" (o en abrir-la-reserva.vbs).
     Espera unos segundos: se abre el navegador en http://localhost:8080
     (sin ventanas negras de CMD). El servidor sigue en segundo plano.

  IP PARA CELULARES (admin)
  -------------------------
  • En la app: Mesas → icono Conexion movil.
    Muestra codigo QR y URL para abrir en el navegador del celular (misma Wi-Fi).

  OPCIÓN B — Con ventanas CMD (para ver errores al instalar o depurar)
  1–3. Igual que arriba.
  4. Doble clic en inicio.bat
  5. Deja minimizadas las ventanas LaReserva_API y LaReserva_Web.

  Desde otro dispositivo en la misma red Wi-Fi:
     http://IP_WIFI_DE_ESTE_PC:8080
     (La IP y el QR estan en admin - Conexion movil. NO uses 192.168.56.x.)

  Si antes veías puertos raros (61996, 59191): el 8080 estaba ocupado. Tras
  actualizar inicio.bat se usa puerto fijo 8080; libéralo o cambia WEB_PORT.

  Comprueba el API desde el celular:  http://IP_WIFI:3000/health
  Si la web carga pero no el login → bin\abrir-firewall.bat (administrador).


WI-FI SIN INTERNET (solo red local)
-----------------------------------
  La Reserva NO necesita internet mundial: el PC hace de servidor y los celulares
  entran por la misma Wi‑Fi (http://IP_PC:8080 → API en :3000).

  Si el router pierde internet y la app deja de funcionar:
  • El servidor web ya no usa npx (no descarga nada de internet al arrancar).
  • En los celulares Android: si aparece "Conectado, sin internet", desactiva
    "Cambiar a datos móviles" / "Asistencia de red" para esa Wi‑Fi y mantén
    los datos móviles apagados mientras trabajas (evita que el navegador salga
    por 4G en vez de la LAN).
  • Comprueba desde el celular: http://IP_PC:3000/health  y  http://IP_PC:8080
  • Si /health responde pero la app no: reexporta la web (empaquetar-web) para
    tener la versión que detecta la red local sin Google.


DETENER
-------
  Sin ventana: bin\detener.vbs
  Con confirmación en CMD: detener.bat
  Cerrar solo el navegador NO apaga el servidor para los celulares.


SI ALGO FALLA EN MODO SILENCIOSO
--------------------------------
  Revisa api\logs\setup.log y api\logs\stderr.log
  O usa inicio.bat una vez para ver el error en pantalla.


LA APP SE CAE PARA TODOS AL DEJAR EL PC QUIETO
----------------------------------------------
  Causas habituales y qué hacer:

  1. Windows suspende el PC por inactividad
     → Ejecuta bin\evitar-suspension.bat (una vez).
     → Panel de control → Energía → Suspensión: Nunca (con corriente).

  2. Se cerraron las ventanas LaReserva_API o LaReserva_Web
     → Vuelve a ejecutar inicio.bat y déjalas minimizadas, no las cierres.

  3. Se cerró sesión en Windows (no solo bloquear pantalla)
     → Usa bloqueo de pantalla; no «Cerrar sesión».

  4. PostgreSQL dejó de correr
     → Servicios de Windows → postgresql → Inicio automático.

  5. El API crasheó (error en consola LaReserva_API)
     → Desde inicio.bat nuevo, el supervisor reinicia solo en ~3 s.
     → Si se repite: revisa DATABASE_URL y que PostgreSQL responda.

  Comprobar desde un celular:
    http://IP_PC:3000/health   → debe responder {"status":"ok"} o similar
    http://IP_PC:8080          → debe cargar la app


ACTUALIZAR EN EL MISMO PC
-------------------------
  1. detener.bat
  2. Sustituye la carpeta LaReserva por la nueva (o solo api\ + web\).
  3. NO borres api\.env ni api\license.key (configuración y licencia de este PC).
  4. inicio.bat aplicará las migraciones nuevas sobre la misma base de datos.


ACTUALIZAR EN OTRO PC (carpeta nueva)
-------------------------------------
  1. Copia LaReserva\ completa (incluye api\node_modules, vendor, dist, web\).
  2. Crea/edita api\.env con la DATABASE_URL de ESE PC.
  3. Solicita una licencia NUEVA para ese PC (bin\mostrar-id-maquina.bat).
     La license.key del PC anterior NO sirve en otro equipo.
  4. inicio.bat crea el esquema si la base está vacía, o migra si ya existía.
  5. Si borras LaReserva\ y la vuelves a copiar, repite pasos 2-4 (la base queda en PostgreSQL).

  IMPORTANTE al copiar:
  • NO copies solo api\dist (falta vendor\ y node_modules).
  • Si usas USB/ZIP, incluye api\vendor\ y api\node_modules\ (son grandes pero necesarios).
  • Conserva api\.env y api\license.key del restaurante; no los sobrescribas.


Error MODULE_NOT_FOUND (@la-reserva/shared-domain / usuario-display)
--------------------------------------------------------------------
  El API arrancó sin la librería compartida empaquetada. En el PC del restaurante
  deben existir estas rutas:

    api\vendor\shared-domain\dist\index.js
    api\node_modules\@la-reserva\shared-domain\

  Solución rápida (en el PC del restaurante):
    1. detener.bat
    2. Copia de nuevo la carpeta api\ COMPLETA desde tu PC de desarrollo
       (después de npm run la-reserva:empaquetar), pero NO reemplaces api\.env ni api\license.key
    3. Si ya tienes api\vendor\, ejecuta bin\reparar-api.bat y luego inicio.bat

  Si solo copiaste dist\ encima de una instalación vieja, eso provoca este error.


APP ANDROID (APK)
-----------------
  La URL del API va embebida en el APK al compilar. Debe coincidir con la IP del
  PC donde corre el servidor (ej. http://192.168.1.7:3000). Si cambias de PC o IP,
  hay que generar de nuevo el APK con esa URL.


FIREWALL DE WINDOWS
-------------------
  Si no entra desde los celulares, ejecuta bin\abrir-firewall.bat (acepta admin)
  o permite manualmente TCP entrante en puertos 3000 (API) y 8080 (web).


Error Prisma P3005 (base no vacía / baseline)
--------------------------------------------
  Suele aparecer si la base ya tenía tablas creadas con "db push" u otra vía,
  pero Prisma Migrate no tiene el historial. Solución (una sola vez):

    bin\ejecutar-baseline-prisma.bat

  Luego vuelve a usar inicio.bat. No uses el baseline en una base nueva y vacía.


Error Prisma P1000 (autenticación)
----------------------------------
  Significa que usuario o contraseña en api\.env → DATABASE_URL no coinciden con
  PostgreSQL. No uses literales como USER:PASSWORD: pon el usuario real (casi
  siempre "postgres" en Windows) y la contraseña que definiste al instalar Postgres.
  Crea antes la base de datos "la_reserva" en pgAdmin. Si la contraseña tiene
  caracteres especiales, búscalos codificados para URL (p. ej. @ → %40).


SOPORTE TÉCNICO
---------------
  Problemas típicos:
  • "migrate deploy" falla → PostgreSQL no arrancado o DATABASE_URL incorrecta.
  • La web no carga → revisa que exista la carpeta web\ con el export de Expo.
  • El APK no conecta → misma Wi-Fi que el PC y EXPO_PUBLIC_API_URL correcta en el build.
