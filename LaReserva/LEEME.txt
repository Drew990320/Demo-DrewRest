================================================================================
  LA RESERVA — Instalación en el PC del restaurante
================================================================================

CONTENIDO DE ESTA CARPETA
-------------------------
  api\          Servidor NestJS compilado (dist, node_modules, vendor, prisma, package.json)

  Si falta api\dist\main.js:
  • En el equipo de desarrollo:  npm run la-reserva:empaquetar
    (compila API + web y deja LaReserva\ listo para copiar al restaurante).
  • En el PC del cliente: deben copiarte la carpeta LaReserva\ completa.
  web\          Archivos estáticos de la app web (export Expo: index.html, _expo, etc.)

  Si falta web\index.html:
  • En desarrollo:  powershell -ExecutionPolicy Bypass -File scripts\empaquetar-web-la-reserva.ps1
    (usa EXPO_PUBLIC_API_URL de apps\mobile\.env o http://192.168.1.7:3000 por defecto).
  apk\          Instalable Android para meseros/cocina (opcional)
  inicio.bat                    Doble clic para arrancar API + web (muestra IP al inicio y al final)
  mostrar-ip.bat                Solo la IP para el celular (Wi-Fi o Ethernet, sin redes virtuales)
  ejecutar-baseline-prisma.bat  Solo si migrate falla con P3005 (ver arriba)
  detener.bat                   Cierra el API y el servidor web
  detener.ps1   Ayuda usada por detener.bat (no hace falta ejecutarlo a mano)
  LEEME.txt     Este archivo


REQUISITOS (solo una vez en el PC)
----------------------------------
  • PostgreSQL instalado y con una base creada (ej. base "lareserva").
  • Node.js LTS instalado desde https://nodejs.org (siguiente → siguiente).


CONFIGURACIÓN DEL API
---------------------
  En la carpeta api\ debe existir un archivo .env con, como mínimo:

    DATABASE_URL="postgresql://USUARIO:CONTRASEÑA@localhost:5432/NOMBRE_BD?schema=public"
    JWT_SECRET="una-cadena-larga-y-aleatoria"
    PORT=3000
    HOST=0.0.0.0
    NODE_ENV=production

  Ajusta USUARIO, CONTRASEÑA y NOMBRE_BD a tu PostgreSQL.
  HOST=0.0.0.0 permite que celulares en la Wi-Fi lleguen al API usando la IP del PC.

IMPRESORA COMANDA (opcional, Jaltech 58M u otra ESC/POS 58 mm)
----------------------------------------------------------------
  Conecta la impresora al PC del restaurante (USB o Bluetooth emparejado).
  En api\.env agrega (nombre exacto como aparece en Windows):

    PRINTER_ENABLED=true
    PRINTER_INTERFACE=printer:NombreDeTuImpresora
    PRINTER_WIDTH=32

  Si Bluetooth usa puerto COM:  PRINTER_INTERFACE=COM3,COM4
  Si USB (nombre en Windows):   PRINTER_INTERFACE=printer:POS-58
  Prueba desde services/api:    npx tsx scripts/test-print-comanda.ts
  El mesero pulsa «Pasar a cocina» en la app; el API imprime solo platos
  (sin bebidas ni precios). El cobro final incluye todo el pedido.

USUARIOS INICIALES (AUTOMÁTICO)
-------------------------------
  En cada inicio, el sistema verifica datos base sin borrar nada:
  - Roles: mesero, chef, admin
  - Mesas: 1-15, 98 y 99
  - Usuarios demo (solo si no existen):
      mesero@lareserva.local / mesero123
      chef@lareserva.local   / chef123
      admin@lareserva.local  / admin123


ARRANQUE
--------
  1. PostgreSQL en marcha y base creada (ej. la_reserva).
  2. Edita api\.env con DATABASE_URL y JWT_SECRET (conserva este archivo al
     actualizar LaReserva en el mismo PC).
  3. Doble clic en inicio.bat
  4. El .bat aplica migraciones pendientes, verifica datos base e inicia API+web.
  5. Abre en el navegador la URL que muestra inicio.bat (puerto 8080, o el siguiente libre si 8080 esta ocupado p. ej. por Postgres EDB).

  Desde otro dispositivo en la misma red Wi-Fi:
     http://IP_WIFI_DE_ESTE_PC:8080
     (Ejecuta mostrar-ip.bat o ipconfig → adaptador "Wi-Fi". NO uses 192.168.56.x.)

  Si antes veías puertos raros (61996, 59191): el 8080 estaba ocupado. Tras
  actualizar inicio.bat se usa puerto fijo 8080; libéralo o cambia WEB_PORT.

  Comprueba el API desde el celular:  http://IP_WIFI:3000/health
  Si la web carga pero no el login → abrir-firewall.bat (administrador).


WI-FI SIN INTERNET (solo red local)
-----------------------------------
  La Reserva NO necesita internet mundial: el PC hace de servidor y los celulares
  entran por la misma Wi‑Fi (http://IP_PC:8080 → API en :3000).

  Si el router pierde internet y la app deja de funcionar:
  • El servidor web ya no usa npx (no descarga nada de internet al arrancar).
  • En los celulares Android: si aparece "Conectado, sin internet", desactiva
    "Cambiar a datos móviles" / "Asistencia de red" para esa Wi‑Fi y mantén
    los datos móviles apagados mientras trabajas (evita que el navegador salga
    por 4G en vez de la LAN).
  • Comprueba desde el celular: http://IP_PC:3000/health  y  http://IP_PC:8080
  • Si /health responde pero la app no: reexporta la web (empaquetar-web) para
    tener la versión que detecta la red local sin Google.


DETENER
-------
  Doble clic en detener.bat, o cierra las dos ventanas de consola (API y Web).


ACTUALIZAR EN EL MISMO PC
-------------------------
  1. detener.bat
  2. Sustituye la carpeta LaReserva por la nueva (o solo api\ + web\).
  3. NO borres api\.env (tiene tu PostgreSQL y JWT).
  4. inicio.bat aplicará las migraciones nuevas sobre la misma base de datos.


ACTUALIZAR EN OTRO PC (carpeta nueva)
-------------------------------------
  1. Copia LaReserva\ completa (incluye api\node_modules, vendor, dist, web\).
  2. Crea/edita api\.env con la DATABASE_URL de ESE PC.
  3. inicio.bat crea el esquema si la base está vacía, o migra si ya existía.
  4. Si borras LaReserva\ y la vuelves a copiar, repite pasos 2-3 (la base queda en PostgreSQL).


APP ANDROID (APK)
-----------------
  La URL del API va embebida en el APK al compilar. Debe coincidir con la IP del
  PC donde corre el servidor (ej. http://192.168.1.7:3000). Si cambias de PC o IP,
  hay que generar de nuevo el APK con esa URL.


FIREWALL DE WINDOWS
-------------------
  Si no entra desde los celulares, ejecuta abrir-firewall.bat (acepta admin)
  o permite manualmente TCP entrante en puertos 3000 (API) y 8080 (web).


Error Prisma P3005 (base no vacía / baseline)
--------------------------------------------
  Suele aparecer si la base ya tenía tablas creadas con "db push" u otra vía,
  pero Prisma Migrate no tiene el historial. Solución (una sola vez):

    ejecutar-baseline-prisma.bat

  Luego vuelve a usar inicio.bat. No uses el baseline en una base nueva y vacía.


Error Prisma P1000 (autenticación)
----------------------------------
  Significa que usuario o contraseña en api\.env → DATABASE_URL no coinciden con
  PostgreSQL. No uses literales como USER:PASSWORD: pon el usuario real (casi
  siempre "postgres" en Windows) y la contraseña que definiste al instalar Postgres.
  Crea antes la base de datos "la_reserva" en pgAdmin. Si la contraseña tiene
  caracteres especiales, búscalos codificados para URL (p. ej. @ → %40).


SOPORTE TÉCNICO
---------------
  Problemas típicos:
  • "migrate deploy" falla → PostgreSQL no arrancado o DATABASE_URL incorrecta.
  • La web no carga → revisa que exista la carpeta web\ con el export de Expo.
  • El APK no conecta → misma Wi-Fi que el PC y EXPO_PUBLIC_API_URL correcta en el build.
